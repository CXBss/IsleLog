## 1. 项目概述

本项目旨在开发一款跨平台的日记/备忘录应用。该应用使用 **Flutter** 框架开发，采用  **“离线优先 (Offline-first)”**  架构。

- **本地存储**：拥有完整的本地数据库，即使在无网络或未连接服务器的情况下，用户也能顺畅地进行查看、增删改查日记操作。
- **云端同步**：使用开源项目 [Memos](https://github.com/usememos/memos) 的 OpenAPI 作为线上同步后端。当网络恢复时，应用会在后台自动或手动与 Memos 服务器进行双向同步。

## 2. 技术栈建议

- **前端框架**：Flutter
- **本地数据库**：Isar
- **网络请求**：Dio (用于与 Memos API 交互)
- **路由管理**：GoRouter

## 3. 核心功能与架构要求

### 3.1 离线优先与同步逻辑

- **数据模型 (Data Model)** ：每条日记需要包含但不限于以下字段：`id` (本地主键), `memos_id` (远端Memos的ID), `content` (正文), `created_at` (创建时间), `tags` (标签), `location` (位置), `sync_status` (同步状态: 待同步/已同步/有冲突)。
- **写入逻辑**：用户新建或修改日记时，**首先且必须**写入本地数据库，然后界面立即响应。随后在后台异步尝试推送到 Memos 服务器。如果失败，保留 `待同步` 状态，等待下次网络恢复。
- **拉取逻辑**：应用启动或下拉刷新时，根据最后同步时间 (last\_sync\_time) 从 Memos 拉取增量更新，并合并到本地数据库。

### 3.2 UI 视图与交互 (参考设计图)

**全局 UI 元素：**

- **底部导航栏 (Bottom Navigation Bar)** ：包含至少两个核心 Tab：“主页(时间线)” 和 “日历”。
- **悬浮按钮 (FAB)** ：在主页和日历页均常驻一个居中的显眼悬浮按钮 (绿色带 '+' 号)，用于快速发布新日记。

**关于标签的处理逻辑：**

- **输入规范**：兼容 Memos 习惯，标签直接写在 `content` 正文内（如 `#Tag`）。
- **存储逻辑**：在保存到 Isar 数据库前，需自动解析正文提取出标签列表，并存入单独的 `List<String> tags` 字段。
- **索引要求**：对 `tags` 字段建立索引，以便实现高效的标签聚合统计和分类筛选，避免全量文本扫描。

#### 页面 A：时间线视图 (主页 - Home View)

*参考：类似传统手帐或社交媒体的时间线展示。*

- **布局结构**：

    - 左侧显示日期和星期（如：`11 / 12月 星期四`）。
    - 右侧主体区域使用垂直时间线 (Vertical Timeline) 样式连接当天的所有记录。
    - 每条记录左侧有一个时间戳圆点节点 (如 `14:06`, `10:33`)。
- **日记卡片内容**：

    - **正文内容**：支持 Markdown 格式的纯文本渲染。
    - **位置信息**：带有一个定位小图标及位置文本 (如 `江苏省-苏州市...`)，字体较小，呈灰色或蓝色。
    - **标签系统**：以圆角药丸状 (Chip) 展示该条日记的标签 (如 `#想法`, `#生活记录`)，背景色通常为浅色/浅绿色。
    - **操作区**：每条日记底部有菜单按钮(用于编辑/删除)和评论/互动按钮。

#### 页面 B：日历视图 (Calendar View)

*参考：按月查看视角的日历。*

- **顶部控制区**：显示当前年月 (如 `12月 2025`)，并有“回到今日”和“分享月历”等功能按钮。
- **日历主体 (Month View)** ：

    - 标准 7 列布局 (周一至周日)。
    - 日期数字下方需支持显示农历 (如 `十二`, `廿二` 等)。
    - **有记录的日期提示**：如果当天有日记，日期背景需高亮显示 (如浅绿色背景圆角矩形)，或者直接在日期上方显示当天日记包含的首张图片缩略图。
- **日历下方列表**：

    - 点击日历上的某一天，下方列表应平滑切换并展示该选定日期的所有日记条目。
    - UI 样式与“时间线视图”中的单日卡片保持一致。


## 4. 开发步骤拆解

为了保证代码质量，请严格按照以下步骤进行开发，每完成一步后再进行下一步：

1. **第一步：搭建基础项目与本地数据库**

    - 初始化 Flutter 项目。
    - 定义 `MemoEntry` 数据模型。
    - 配置本地数据库 (Isar/SQLite) 并实现基础的 CRUD (增删改查) 接口。暂时不接入网络。
2. **第二步：实现 UI 静态页面与状态绑定**

    - 实现带底部导航栏的主骨架 (Scaffold)。
    - 利用假数据 (Mock Data) 实现**时间线视图 (主页)** 。需重点还原左侧时间线UI、位置和标签Chip。
    - 利用假数据实现**日历视图**。需集成第三方日历组件 (如 `table_calendar`)，实现农历显示和特定日期高亮（带图片/背景色），以及点击日期联动下方列表的功能。
3. **第三步：Memos API 接入与同步机制**

    - 配置 Memos API 的 Base URL 和 Token 认证。
    - 实现 Memos OpenAPI 的网络请求服务。
    - 编写同步逻辑引擎 (Sync Engine)：实现“本地优先写入 -\> 异步推送到远端 -\> 拉取远端更新 -\> 解决简单冲突”。
4. **第四步：细节打磨**

    - 完善新建日记的表单页面。
    - 处理网络异常、Loading 状态和空数据状态的 UI 提示。
